# 测试计划 · Quota Butler V2

## 测试原则

- 群内交互必须在飞书 `额度管家提醒群` 中实测。
- 单元测试覆盖纯逻辑与渲染结构。
- 按钮闭环和主动推送必须有端到端验收。
- 文字 router 是兜底路径，不替代 bridge 按钮测试。

- 用户输入场景的长度限制和 Markdown 转义。

## T1 · Bridge 按钮闭环测试

### 前置条件

- 私人 bridge fork 已构建并以 `codex` profile 常驻。
- launchd 的执行路径指向 `lark-channel-bridge-quota`。

```bash
node bin/lark-channel-bridge.mjs status --profile codex
```

### 用例

| 用例 | 操作 | 预期 |
|------|------|------|
| 菜单按钮 - 明日计划 | 点击“帮我安排明天” | 群内自动出现计划卡 |
| 菜单按钮 - 冲刺 | 点击“今天冲刺” | 群内自动出现 sustain 计划卡 |
| 菜单按钮 - 当前额度 | 点击“当前额度” | 群内自动出现状态卡 |
| 未知 action | 手动调用 handler 传未知 action | 退出码 1，不执行副作用 |
| bridge 停止 | 关闭 bridge 后点按钮 | 飞书提示服务未在线，符合预期 |

### 通过标准

- 不需要手动运行 `chat_router --watch`。
- 点击按钮 5 秒内有群内回复。
- handler 日志能看到对应 action。

## T3 · 计划卡时间线测试

### 单元测试

| 用例 | 配置 | 预期 |
|------|------|------|
| 单 Agent sustain | cc, 8h | 预热提前一个窗口 |
| 双 Agent sustain | cc,codex, 10h | 两 Agent 交错恢复 |
| balanced | cc,codex, 8h | 预热提前半窗口 |
| savings | cc,codex, 8h | 预热提前较少 |
| 睡觉时间截断 | 20:00 开始，23:00 睡 | work_end 不超过睡觉时间 |
| 逐步规划默认值 | 未修改时间 | 北京时间 09:00–17:00 |
| 轻量强度 | 8h | 不设置固定接力点 |
| 正常强度 | 8h | 1 个建议接力点 |
| 高强度 | 8h | 3 个建议接力点 |
| 旧计划 | 无 `plan_version: 2` | 拒绝采用，要求重新规划 |
| 旧引导卡 | `flow_version: 2` | 拒绝继续，要求重新规划 |
| 手机四选项 | 任务类型卡 | 两行两列，按钮文字完整 |
| 首次用户 | 无个人场景配置 | 先出现手动输入场景表单 |
| 返回用户 | 已保存个人场景 | 跳过首次表单，直接进入任务类型 |
| 多用户 | Alice 已配置、Bob 未配置 | Bob 仍看到首次场景表单 |

### 群内测试

发送：

```text
帮我安排明天
今天冲刺
节省模式
```

预期：

- 先依次出现任务类型、强度、时间和摘要卡。
- 时间步骤使用两个原生时间选择器。
- 最终卡显示实际使用的 Agent。
- 最终卡显示计划覆盖率、预计空档和预计接力次数，不显示 `CAS` 缩写。
- 最终卡提供采用计划、调整设置和仅提醒。

## T4 · 采用计划测试

### 前置条件

- Bridge 按钮闭环已通过。
- `采用计划` 已实现状态写入和任务生成。

### 用例

| 用例 | 操作 | 预期 |
|------|------|------|
| 采用计划 | 点击计划卡“采用计划” | 群内回执 active plan |
| 查看计划 | 发送/点击“查看计划” | 返回当前计划卡 |
| 取消计划 | 点击“取消计划” | 删除未执行任务，回执成功 |
| 到点执行 | 等待 warmup 时间 | 自动执行 warmup 并回执 |
| 重复采用 | 已有 active plan 再采用 | 提示覆盖/取消，不静默覆盖 |

### 安全要求

- 真 warmup 前必须明确知道 provider。
- 默认优先 Codex，避免误触 Claude 计费路径。
- 所有生成的 launchd 任务可列出、可删除。

## T5 · 主动 one-up 推送测试

### 前置条件

- 可构造 fake usage 或 mock provider。
- active plan 状态可控。

### 单元测试

| 场景 | 预期 |
|------|------|
| Agent 恢复 + 无 active plan | 推 one-up 卡 |
| Agent 恢复 + 有 active plan | 不推 |
| quiet_hours 内 | 不推 |
| 同窗口已推过 | 不重复推 |
| 用户选择今天不提醒 | 当天不再推 |

### 群内测试

通过 fake provider 或调试配置模拟恢复：

```text
Codex 已恢复，可以 one up 了
[现在启动] [稍后提醒] [今天不提醒]
```

预期：

- 卡片主动出现。
- 点击“现在启动”进入 warmup 流程。
- 点击“稍后提醒”写入 snooze。
- 点击“今天不提醒”写入当天屏蔽状态。
