Quota Butler

我们做这个功能

把 Claude Code 和 Codex 的额度状态、预热计划、完成提醒，收进一个飞书私聊机器人里。

Pain Point

不是额度不够，而是额度总在错误的时间恢复。

看不见

5 小时窗口、周额度、刷新时间分散在不同入口，真正要工作时才发现快用完。

容易错过

凌晨恢复、早上开工、下午接力，各自都需要记时间，靠人脑很难稳定。

切换成本高

Claude Code 和 Codex 都能用，但谁先上、谁接力、什么时候预热，需要系统帮忙算清楚。

How It Works

一个私聊入口，四个核心动作。

1
查询额度直接看还剩多少、什么时候刷新。
2
立即预热手动触发一次真实预热，不创建长期计划。
3
设置明日计划只选开始时间，系统自动安排预热节点。
4
查看当前计划展示未执行、已执行、失败，并可取消所有计划。
