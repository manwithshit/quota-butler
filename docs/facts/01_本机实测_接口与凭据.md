# 🔬 能力实测报告（本机真实凭据验证）

> 这份是我相对云端调研（段誉/GLM）的核心差异：**不是抄端点，是在懒人哥本机用真实凭据把接口打通了。**
> 安全说明：全程 token 只留在 shell 内存变量，**未打印、未外传、未写盘**。下面只贴脱敏结构和响应。

## 一、Claude Code 额度感知 —— 🟢 实测 200 跑通

### 1. 凭据读取（重要纠偏：在 Keychain，不在文件）

本机 **`~/.claude/.credentials.json` 不存在**。CC 在 macOS 把凭据存进钥匙串：

```bash
security find-generic-password -s "Claude Code-credentials" -w
```

返回 JSON 结构（脱敏）：
```json
{
  "claudeAiOauth": {
    "accessToken": "<108 chars>",
    "refreshToken": "<108 chars>",
    "expiresAt": 1781310179507,
    "scopes": ["user:file_upload","user:inference","user:mcp_servers","user:profile","user:sessions:claude_code"],
    "subscriptionType": "pro",
    "rateLimitTier": "default_claude_ai"
  }
}
```

### 2. 实测请求

```bash
curl -s \
  -H "Authorization: Bearer $CC_TOKEN" \
  -H "anthropic-beta: oauth-2025-04-20" \
  -H "Accept: application/json" \
  -H "User-Agent: claude-code/2.1.160" \
  "https://api.anthropic.com/api/oauth/usage"
```

### 3. 实测响应（HTTP 200，你的真实数据）

```json
{
  "five_hour":  {"utilization": 87.0, "resets_at": "2026-06-12T20:50:00.903592+00:00"},
  "seven_day":  {"utilization": 61.0, "resets_at": "2026-06-13T12:00:00.903613+00:00"},
  "seven_day_oauth_apps": null, "seven_day_opus": null, "seven_day_sonnet": null,
  "seven_day_cowork": null, "seven_day_omelette": null,
  "extra_usage": {"is_enabled": false, "monthly_limit": null, "used_credits": null, "utilization": null, "currency": null, "disabled_reason": null}
}
```

### 4. 字段语义（可直接拿来写工具）

| 字段 | 含义 | 用途 |
|------|------|------|
| `five_hour.utilization` | 5h 滚动窗口已用百分比（0–100） | 判断「还剩多少没用」 |
| `five_hour.resets_at` | 5h 窗口重置时间（ISO8601 带时区） | 算「还有多久刷新」 |
| `seven_day.utilization` / `resets_at` | 7天窗口 | 长周期监控 |
| `extra_usage.is_enabled` | 是否开了超额付费 | 你目前 false |

> **关键纠偏**：`resets_at` 是后端**算好的绝对时间戳**，不需要自己根据「首条消息 + 5h」推。直接读即可，比 5/30 待办里「读 jsonl 自己算窗口」精确得多。

### 5. 凭据有效期注意

`expiresAt: 1781310179507`（毫秒）= access_token 有效期。工具要处理过期：CC CLI 在用时自动刷新 Keychain 里的 token；如果工具独立跑、token 过期，需要 fallback（见下方 Codex 同款问题）。

## 二、Codex 额度感知 —— 🟡 端点确认，token 待刷新

### 1. 凭据结构

`~/.codex/auth.json`（脱敏）：
```json
{
  "auth_mode": "chatgpt",
  "OPENAI_API_KEY": null,
  "tokens": {
    "id_token": "<2000 chars JWT>",
    "access_token": "<1975 chars JWT>",
    "refresh_token": "<90 chars>",
    "account_id": "1c512d31-...-c9ed76bd304d"
  },
  "last_refresh": "2026-05-26T07:35:53Z"
}
```

### 2. 实测请求 + 响应

```bash
curl -H "Authorization: Bearer $CX_TOKEN" \
     -H "chatgpt-account-id: $CX_ACCT" \
     "https://chatgpt.com/backend-api/wham/usage"
```
返回 **HTTP 401**：
```json
{"error":{"message":"Provided authentication token is expired. Please try signing in again.","code":"token_expired"},"status":401}
```

### 3. 解读：这其实是好消息

- 解 JWT 看到 `access_token exp = 2026-06-05T07:35:54Z`，**6/5 就过期了**（你最近没用 Codex CLI）。
- 401 是**规整的业务错误**而非「端点不存在」→ **证明端点真实、鉴权头对**（`Authorization: Bearer` + `chatgpt-account-id`）。
- 拿到 fresh token 即可返回真实 quota。

### 4. Codex 响应结构（来自 steipete/CodexBar 权威实现）

```
rate_limit.primary_window      → 5h 主窗口（session lane）
rate_limit.secondary_window    → 7天次窗口（weekly lane）
additional_rate_limits[]       → 按模型/code-review 的细分限额
```
另有 `https://chatgpt.com/backend-api/codex/usage` 也返回各窗口利用率+重置时间。

### 5. Token 刷新策略（工具必须实现）

- access_token **约 1 小时过期**，Codex CLI 在用时自动刷新；`last_refresh > 8 天` 也会触发刷新。
- **最省事方案**：工具不自己重实现 OAuth 刷新，而是**跑一条 `codex exec "ping"`**——既刷新了 token（写回 auth.json），又顺便完成了 warm-up（一箭双雕）。
- 备选：用 `refresh_token` 走 OpenAI OAuth token endpoint 自己刷（需要 Codex 的 client_id，未在本机验证，不建议first版做）。

## 四、给工具的「能力①」最小实现伪码

```python
# CC：读 Keychain → 打 usage
cc_raw = sh('security find-generic-password -s "Claude Code-credentials" -w')
cc_token = json.loads(cc_raw)["claudeAiOauth"]["accessToken"]
cc = GET("https://api.anthropic.com/api/oauth/usage",
         headers={"Authorization": f"Bearer {cc_token}",
                  "anthropic-beta": "oauth-2025-04-20"})
# cc["five_hour"]["utilization"], cc["five_hour"]["resets_at"]

# Codex：读 auth.json → 打 usage，401 则先 `codex exec` 刷新再重试
cx_auth = json.load(open("~/.codex/auth.json"))
cx = GET("https://chatgpt.com/backend-api/wham/usage",
         headers={"Authorization": f"Bearer {cx_auth['tokens']['access_token']}",
                  "chatgpt-account-id": cx_auth["tokens"]["account_id"]})
if cx.status == 401:  # token_expired
    sh('codex exec "ok"')           # 顺便 warm-up + 刷新
    cx_auth = json.load(open("~/.codex/auth.json"))  # 重读
    cx = GET(... 用新 token ...)
```
