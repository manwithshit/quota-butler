# Quota Butler V2 逐步规划交互设计

## 目标

把“帮我安排明天”从直接生成技术计划，改为先采集偏好、再生成人话计划的两阶段交互。用户不再看到难以理解的 CAS 缩写、裸露的预热窗口或无上下文的等待时间，也不能采用旧版计划卡。

## 第一阶段：逐步引导卡

`schedule_intent` 只启动规划流程，不立即生成计划。流程依次收集：

1. 任务类型：编码开发、内容创作、调研分析、混合任务。
2. 工作强度：轻量、正常、高强度。
3. 工作时间：两个飞书原生 `picker_time`，默认北京时间 09:00–17:00。
4. 偏好摘要：展示已选任务、强度、工作时段，允许生成计划或分别修改三项。

每次 callback 都携带完整、可验证的短流程上下文：

```json
{
  "cmd": "quota",
  "action": "schedule_flow",
  "flow_version": 3,
  "step": "intensity",
  "target_date": "2026-06-19",
  "preferences": {
    "task_type": "coding",
    "intensity": "normal",
    "work_start": "09:00",
    "work_end": "17:00"
  }
}
```

流程偏好不写入长期状态。旧版本或过期日期的流程卡只返回“请重新规划”，不继续生成计划。

## Agent 自动选择

用户不选择 Agent。点击“生成计划”时读取 Codex 和 Claude Code 当前状态：

- 任意一个可用：使用可用 Agent 生成计划。
- 两个都可用：自动组合接力。
- 两个都不可用：只说明暂时无法生成计划，不附带登录或恢复建议。

计划卡只展示实际参与的 Agent，不展示其他 Agent 的失败原因。采用计划时再次检查计划内 Agent，避免生成后状态发生变化。

## 第二阶段：人话计划卡

卡片开头直接说明工作时间、实际使用的 Agent 和接力次数，例如：

> 明天 09:00–17:00，主要使用 Codex，共安排 2 次接力。

每个时间节点解释工作目的。任务类型决定节点文案，工作强度决定接力密度：

- 轻量：减少打断，保留较长连续工作段。
- 正常：适度安排接力。
- 高强度：优先缩短潜在空档。

卡片展示增强信任的计算依据，但不出现 `CAS` 缩写：

- 计划覆盖率
- 预计空档
- 预计接力次数
- “按当前额度窗口估算，实际恢复时间可能变化”

操作按钮为：

- 采用计划
- 调整设置
- 仅提醒

“调整设置”返回偏好摘要卡，再由用户选择修改任务、强度或时间。

## 计划版本与安全边界

新计划记录带 `plan_version: 2` 和结构化 `preferences`。`adopt_schedule` 必须拒绝没有该版本标识的旧计划，防止用户采用当前语义不合格的旧卡。

工作时间要求：

- 结束时间晚于开始时间。
- 单次工作时段不超过 16 小时。
- 时间按回调中的用户时区解释；本产品卡片明确显示北京时间。

真实预热继续限制为 Codex。规划和 dry-run 不执行预热，不调用 `claude -p`。实现不读取、打印或提交 App Secret，不修改当前 bridge fork 的 launchd 服务，也不启动第二个飞书 listener。

## 模块边界

- `quota_butler/schedule_flow.py`：纯函数，负责偏好规范化、流程 payload、时间校验和文案映射。
- `quota_butler/planner.py`：接收结构化偏好和可用 Agent，输出带计划版本、覆盖指标及接力事件的计划。
- `quota_butler/notify.py`：渲染逐步引导卡、摘要卡和人话计划卡。
- `quota_butler/handler.py`：编排 callback 流程、Agent 可用性检查和旧计划拦截。
- `quota_butler/schedule.py`：保留 CLI dry-run，并使用默认偏好直接生成可检查的新计划卡。

## 测试策略

严格按 TDD 实施：

1. 旧计划不能采用。
2. 任务类型和工作强度卡传递完整流程上下文。
3. 时间选择表单解析、默认值和错误边界。
4. Agent 自动选择：单 Agent、双 Agent、全部不可用。
5. 强度映射到接力密度。
6. 任务类型映射到人话节点说明。
7. 新计划卡展示覆盖率、空档、接力次数和三个操作按钮。
8. 全量单元测试与 CLI dry-run 卡片 JSON 检查。
9. 自动化通过后，再保留当前单 listener 进行飞书群内逐按钮验收。
